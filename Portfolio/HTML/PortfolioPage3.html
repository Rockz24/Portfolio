CRUD operation in PythonProject

Click me to access on GITHUB

In this project I developed a command-line Python application for FilmFlix to manage their film database using SQLite3. The application allows for CRUD (Create, Read, Update, Delete) operations on the database, and I have implemented various features such as printing all records, adding, updating, and deleting records, as well as generating and displaying different reports based on SQL commands. This is an important step in helping FilmFlix efficiently manage their film database and make data-driven decisions.

I initiated the process by importing the built-in sqlite3 module into a new Python file (connect.py). Subsequently, I included the connect method, a key component of the sqlite3 module, to establish a connection with the existing database file named 'filmflix.db'. To execute SQL queries and commands, I created a cursor object. This cursor serves the purpose of translating the SQL code, which is necessary because Python does not recognise commands like 'SELECT * FROM' natively. Therefore, the cursor plays a vital role in enabling the execution of such SQL queries within the Python environment.

In a new file named createSQLiteTbls.py, I imported the connect.py module to facilitate the creation of the required table within the database.

I created a new file named addFilms.py with the purpose of adding films to the database. Successfully linking the code from the connect file to the addFilms page, I proceeded to develop a subroutine that enables the addition of films to the database.

I then added a new file called deleteFilms.py to delete records. Here I also imported the connect.py module. After which I developed a subroutine using the filmID(Primary Key) to select and delete a record in the DB table.

I proceeded to create a new file called amendFilms.py to update records inside the database and readFilms.py to observe changes inside our database within the terminal.


Here, I created a new file called menu.py to function as a menu for the user, enabling them to create, read, update, and delete records all on the same page. In order to achieve this, I imported all necessary modules, such as readFilms, addFilms, amendFilms, and deleteFilms. Additionally, I developed another subroutine function, which includes a Boolean variable and utilises if/else/elif statements.

In my final phase, I created a file called reports.py to generate reports based on the user's selection. In this context, I devised several functions to prompt the user for film-related details, including genre, film ID, year, and rating.
